Benchmark functions for optimization.

The Ackley function in its two-dimensional form, it is characterized by a nearly flat outer region, and a large hole at the centre.

dixon_price(x)#

The dixon-price function is unimodal, and the global minimum lies in a narrow, parabolic valley.

float

easom(x)#

The Easom function has several local minima. It is unimodal, and the global minimum has a small area relative to the search space.

michalewicz(x)#

The Michalewicz function has d! local minima, and it is multimodal. The parameter m defines the steepness of they valleys and ridges. A larger m leads to a more difficult search.

metapy_toolbox.ego module#

Efficient Global Optimization (EGO) related functions.

ego_01(obj, n_gen, params, initial_population, x_lower, x_upper, args=None, robustness=False)#

Efficient Global Optimization (EGO) algorithm.

Parameters:

  • obj (Callable) – The objective function: obj(x, args) -> float or obj(x) -> float, where x is a list with shape dim and args is a tuple fixed parameters needed to completely specify the function

  • n_gen (int) – Number of generations or iterations

  • params (dict) – Parameters of Genetic Algorithm

  • initial_population (list) – Initial population

  • x_lower (list) – Lower limit of the design variables

  • x_upper (list) – Upper limit of the design variables

  • robustness (Union[bool, dict]) – If True, the objective function is evaluated in a robust way (default is False)

  • args (Optional[tuple]) – Extra arguments to pass to the objective function (optional)

Returns:

[0] = All evaluations dataframe, [1] = Best, average and worst values dataframe, [2] = Report about the optimization process

metapy_toolbox.funcs module#

Commonly used functions.

best_avg_worst(df, d)#

Check and save best, average and worst objective function values from dataframe

Parameters:

  • df (DataFrame) – Dataframe with all data

  • d (int) – problem dimension

Return type:

DataFrame

Returns:

Positions, Objective function values saved

check_interval_01(x, x_lower, x_upper)#

This function checks if a design variable is out of the limits established x_lower and x_upper and updates the variable if necessary.

Parameters:

  • x (list) – Design variables to be checked

  • x_lower (list) – Lower limit of the design variables

  • x_upper (list) – Upper limit of the design variables

Return type:

list

Returns:

Checked design variables

compare_and_save(df_current, df_temp)#

Compare current and temporary solutions and save the best one.

Parameters:

  • df_current (DataFrame) – Positions, Objective function values, Fitness values and Time consumption saved of the current solution

  • df_temp (DataFrame) – Positions, Objective function values, Fitness values and Time consumption saved of the temporary solution

Return type:

DataFrame

Returns:

Positions, Objective function values, Fitness values and Time consumption saved of the best solution

evaluation(obj, id, x, t, args=None)#

Objective function evaluation and save in dataframe

Parameters:

  • obj (Callable) – The objective function: obj(x, args) -> float or obj(x) -> float, where x is a list with shape dim and args is a tuple fixed parameters needed to completely specify the function

  • id (int) – identifier of the agent

  • x (list) – Design variables to be evaluated

  • neof_count – Count of objective function evaluations

  • t (int) – Current iteration number

  • args (Optional[tuple]) – Extra arguments to pass to the objective function (optional)

Return type:

DataFrame

Returns:

Positions, Objective function values, Fitness values and Time consumption saved

fit_value(of_i_value)#

Calculates the fitness value of the i-th agent based on its objective function value.

Parameters:

of_i_value (float) – Objective function value of the i-th agent in t time step

Return type:

float

Returns:

Fitness value of the i-th agent in t time step (always positive)

mutation_01_random_walk(parent_0, pdf, cov, x_lower, x_upper)#

This function performs the random walk mutation operator. Three new points are generated from the two parent points (offspring).

Parameters:

  • parent_0 (list) – First parent

  • pdf (str) – Probability density function. Options: ‘gaussian’, ‘uniform’ or ‘gumbel’

  • cov (float) – Coefficient of variation

  • x_lower (list) – Lower limit of the design variables

  • x_upper (list) – Upper limit of the design variables

Return type:

tuple[list, str]

Returns:

[0] = First offspring position, [1] = Report about the linear crossover process

query_x_of_fit_from_data(df, i, d)#

Query position, objective function value and fitness value of the i-th agent from dataframe

Parameters:

  • df (DataFrame) – All data in time step t

  • i (int) – Current agent in t time step

  • d (int) – Number of dimensions

Return type:

tuple[list, float, float]

Returns:

[0] = Position of the i-th agent in t time step, [1] = Objective function value of the i-th agent in t time step, [2] = Fitness value of the i-th agent in t time step

Genetic Algorithm related functions.

blxalpha_crossover(parent_0, parent_1, x_lower, x_upper)#

This function performs the BLX-alpha crossover operator. Two new points are generated from the two parent points (offspring).

Parameters:

  • parent_0 (list) – First parent

  • parent_1 (list) – Second parent

  • x_lower (list) – Lower limit of the design variables

  • x_upper (list) – Upper limit of the design variables

Return type:

tuple[list, list, str]

Returns:

[0] = First offspring position, [1] = Second offspring position, [2] = Third offspring position, [3] = Report about the linear crossover process

genetic_algorithm_01(obj, n_gen, params, initial_population, x_lower, x_upper, args=None, robustness=False)#

Genetic algorithm 01. Supports: roulette wheel selection, linear crossover and random walk mutation.

Parameters:

  • obj (Callable) – The objective function: obj(x, args) -> float or obj(x) -> float, where x is a list with shape dim and args is a tuple fixed parameters needed to completely specify the function

  • n_gen (int) – Number of generations or iterations

  • params (dict) – Parameters of Genetic Algorithm

  • initial_population (list) – Initial population

  • x_lower (list) – Lower limit of the design variables

  • x_upper (list) – Upper limit of the design variables

  • robustness (Union[bool, dict]) – If True, the objective function is evaluated in a robust way (default is False)

  • args (Optional[tuple]) – Extra arguments to pass to the objective function (optional)

Return type:

tuple[DataFrame, DataFrame, str]

Returns:

[0] = All evaluations dataframe, [1] = Best, average and worst values dataframe, [2] = Report about the optimization process

linear_crossover(parent_0, parent_1, x_lower, x_upper)#

This function performs the linear crossover operator. Three new points are generated from the two parent points (offspring).

Parameters:

  • parent_0 (list) – First parent

  • parent_1 (list) – Second parent

  • x_lower (list) – Lower limit of the design variables

  • x_upper (list) – Upper limit of the design variables

Return type:

tuple[list, list, list, str]

Returns:

[0] = First offspring position, [1] = Second offspring position, [2] = Third offspring position, [3] = Report about the linear crossover process

roulette_wheel_selection(fit_pop, i_pop)#

This function selects a position from the population using the roulette wheel selection method.

Parameters:

  • i_pop (int) – current agent in t time step
