metapy_toolbox.benchmark module#

benchmark functions for optimization problems

ackley(x, none_variable=None)#

The Ackley function in its two-dimensional form, it is characterized by a nearly flat outer region, and a large hole at the centre.

See the Ackley function documentation.

Parameters:

  • x (List[float]) – List of design variables.

  • none_variable (object | None) – Optional variable for compatibility with general-purpose frameworks.

Returns:

Objective function value.

Return type:

float

dixon_price(x, none_variable=None)#

Dimensions: d

See the Dixon-Price function documentation.

Parameters:

  • x (List[float]) – List of design variables.

  • none_variable (object | None) – Optional variable for compatibility with general-purpose frameworks.

Returns:

Objective function value.

Return type:

float

easom(x, none_variable=None)#

The Easom function has several local minima. It is unimodal, and the global minimum has a small area relative to the search space.

See the Easom function documentation.

Parameters:

  • x (List[float]) – List of design variables (length 2).

  • none_variable (object | None) – Optional variable for compatibility with general-purpose frameworks.

Returns:

Objective function value.

Return type:

float

goldstein_price(x, none_variable=None)#

Dimensions: 2

The Goldstein-Price function has several local minima.

See the Goldstein-Price function documentation.

Parameters:

  • x (List[float]) – List of 2 design variables [x1, x2].

  • none_variable (object | None) – Optional variable for compatibility with general-purpose frameworks.

Returns:

Objective function value.

Return type:

float

griewank(x, none_variable=None)#

The Griewank function has many widespread local minima, which are regularly distributed.

See the Griewank function documentation.

Parameters:

  • x (List[float]) – List of design variables.

  • none_variable (object | None) – Optional variable for compatibility with general-purpose frameworks.

Returns:

Objective function value.

Return type:

float

michalewicz(x, none_variable=None)#

The Michalewicz function has d! local minima, and it is multimodal. The parameter m defines the steepness of they valleys and ridges; a larger m leads to a more difficult search.

See the Michalewicz function documentation.

Parameters:

  • x (List[float]) – List of design variables.

  • none_variable (object | None) – Optional variable for compatibility with general-purpose frameworks.

Returns:

Objective function value.

Return type:

float

powell(x, none_variable=None)#

Dimensions: d

See the Powell function documentation.

Parameters:

  • x (List[float]) – List of design variables. Length should be a multiple of 4.

  • none_variable (object | None) – Optional variable for compatibility with general-purpose frameworks.

Returns:

Objective function value.

Return type:

float

rastrigin(x, none_variable=None)#

The Rastrigin function has several local minima. It is highly multimodal, but locations of the minima are regularly distributed.

See the Rastrigin function documentation.

Parameters:

  • x (List[float]) – List of design variables.

  • none_variable (object | None) – Optional variable for compatibility with general-purpose frameworks.

Returns:

Objective function value.

Return type:

float

return_variance_portifolio(x, mu, sigma)#

This function calculates the annualized return and volatility of a portifolio.

Parameters:

  • x (List[float]) – List of stock weights in the portfolio.

  • mu (List[float]) – List of mean daily returns of the stocks.

  • sigma (List[List[float]]) – Covariance matrix of returns (nested list).

Returns:

Tuple containing:

  • retu: Annualized return of the portfolio (%).

  • voll: Annualized volatility of the portfolio (%).

Return type:

Tuple[float, float]

rosenbrock(x, none_variable=None)#

The Rosenbrock function is unimodal, and the global minimum lies in a narrow, parabolic valley.

See the Rosenbrock function documentation.

Parameters:

  • x (List[float]) – List of design variables.

  • none_variable (object | None) – Optional variable for compatibility with general-purpose frameworks.

Returns:

Objective function value.

Return type:

float

sharp_index(ret, vol, risk_free_asset=0)#

This function calculates the Sharp index.

Parameters:

  • ret (float) – Portfolio return (%).

  • vol (float) – Portfolio volatility (%).

  • risk_free_asset (float) – Risk-free asset return (%).

Returns:

Sharpe index value.

Return type:

float

sphere(x, none_variable=None)#

The Sphere function has d local minima except for the global one. It is continuous, convex and unimodal.

Parameters:

  • x (List[float]) – List of design variables.

  • none_variable (object | None) – Optional variable for compatibility with general-purpose frameworks.

Returns:

Objective function value.

Return type:

float

stock_return_and_covariance(df)#

This function calculates the stock returns.

Parameters:

df (DataFrame) – DataFrame with adjusted close prices of stocks.

Returns:

Tuple containing:

  • df_return: DataFrame with daily returns.

  • mu: List of mean daily returns per stock.

  • sigma: Covariance matrix of returns as nested list.

Return type:

Tuple[DataFrame, List[float], List[List[float]]]

stocks_data(stocks, start_date, end_date)#

This function downloads the stock data from Yahoo Finance.

Parameters:

  • stocks (List[str]) – List of stock tickers (e.g., [‘AAPL’, ‘MSFT’]).

  • start_date (str) – Start date of the historical data in ‘YYYY-MM-DD’ format.

  • end_date (str) – End date of the historical data in ‘YYYY-MM-DD’ format.

Returns:

DataFrame containing adjusted close prices for the selected stocks.

Return type:

DataFrame

zakharov(x, none_variable=None)#

The Zakharov function has no local minima except the global one.

See the Zakharov function documentation.

Parameters:

  • x (List[float]) – List of design variables.

  • none_variable (object | None) – Optional variable for compatibility with general-purpose frameworks.

Returns:

Objective function value.

Return type:

float
